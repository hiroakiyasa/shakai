# 別端末ChatGPTで残り25枚を生成するための起動プロンプト

残り: **25枚**（画像576 と 617〜640）。
`REMAINING_R1`(10枚) → `REMAINING_R2`(10枚) → `REMAINING_R3`(5枚) の順に、**1セッション1バッチ**で実行する。

---

## 使い方

1. ChatGPT（画像生成が使えるモデル）で新しいチャットを開く。
2. 下の「共通プロンプト」の `＜ここにバッチ名＞` を `REMAINING_R1` に置き換えて貼る。
3. 10枚が揃ったら**新しいチャット**を開き、`REMAINING_R2` で同じことを繰り返す。
4. 最後に `REMAINING_R3`（5枚）。

**1チャットで2バッチ以上やらないこと。** 文脈が伸びるとデザインが崩れ、10枚を1枚にまとめる事故が起きる。

---

## 共通プロンプト（これをそのまま貼る）

```
あなたは中学受験「社会」ビジュアル図鑑の画像生成担当です。
GitHubの公開リポジトリ hiroakiyasa/shakai を参照して作業します。

【手順1】まず次のURLをすべて開いて内容を読み込んでください。
https://raw.githubusercontent.com/hiroakiyasa/shakai/main/image_generation/public_batches/＜ここにバッチ名＞.md

そのファイルに「取得すべき raw URL」の一覧があります。
そこに載っているURLを1つ残らず開いてください。内訳は次の4種類です。
 (a) MASTER_PROMPT.md / LOCKED_GENERATION_RULES.md / CURRICULUM_RULES.md / state.json … 守るべき規則
 (b) assets/character_reference.png … 登場キャラクター（宇宙飛行士のヒョウ先生）の姿
 (c) style_reference/ の完成画像8枚 … 再現すべきデザインと品質の見本
 (d) pages_md/ pages/ figures/ tables/ … 今回描く内容の元データ

【手順2】(c)の完成画像8枚を必ず全部見て、次を言語化してから描き始めてください。
 ・配色、余白の取り方、角丸の量、枠線の太さ
 ・タイトルの位置と大きさ、本文の級数バランス
 ・図解の描き方（フラットなベクター調イラスト、細い黒縁、淡い塗り）
 ・重要語句の赤 #D71920 + 太字 + 同色の実線下線の使い方
 ・ヒョウ先生の配置と大きさ
既存616枚と並べて違和感がゼロになることが最優先です。新しいデザインを考えないでください。

【手順3】バッチ表の各行について、1行ずつ画像を生成します。

★最重要ルール（破ったらやり直し）
 ・画像生成ツールを「行数と同じ回数」呼ぶ。R1/R2なら10回、R3なら5回。
 ・1回の呼び出しには1項目だけ。複数項目を1枚にまとめない。
 ・グリッド、コラージュ、タイル、2×5、4分割、コンタクトシートは全部不合格。
 ・サイズは縦9:16、941×1672ピクセルのPNGラスター。SVG・ベクター書き出しは禁止。
 ・生成後にファイル数が行数ちょうどか、各PNGが単一トピックの1ページになっているか自己点検する。

★内容ルール
 ・元の文章・図版・写真・表・誌面をそのまま写さない。事実・数値・関係だけを読み取り、
   独自の図解・地図・年表・統計カードへ描き直す。
 ・元書籍名、出版社名、著者名、ISBN、ロゴは画像内に一切出さない。
 ・地名・人名・年号・制度名・出来事の順序・地図上の位置を正確にする。
   統計は元資料の基準年を明記し、勝手に現在値へ更新しない。
 ・確定できない内容は創作せず、同じページのJSON/Markdownで裏を取る。

★レイアウトルール
 ・大図解1つ ＋ 説明3〜5ブロックまで。8項目以上の一覧や密集した比較表は作らない。
 ・1080×1920換算で本文34px以上、図表ラベル30px以上。
   中学受験の頻出語句・重要数値は48〜64px・太字。
 ・重要語句は #D71920・太字・同色の実線下線。この色は重要語句と下線以外に使わない。
 ・頻出ポイントは対応する図の近くに置く。最下段のまとめ箱に寄せない。
 ・穴埋め、選択肢、クイズ、タップUI、正誤表示は画像内に入れない。
 ・ヒョウ先生は毎回同じ姿。画面面積18%以内。本文・地図・年表を隠さない。

【手順4】1枚できるたびに、そのつど画像を出力してください。
ファイル名は バッチ表の画像番号 + 内容を表す英小文字スラッグ にします。
例: 617_population_statistics.png / 630_index_ho.png
「_auto」という名前は使わないでください。

【手順5】全部終わったら、次の形式で一覧を出してください。
 画像番号 / ファイル名 / 元ページ / 1行要約

まず手順1の読み込み結果を報告してから、生成を始めてください。
```

## バッチ別・置き換える文字列

| 順番 | ＜ここにバッチ名＞ | 枚数 | 対象画像 |
|---|---|---:|---|
| 1回目 | `REMAINING_R1` | 10 | 576, 617〜625 |
| 2回目 | `REMAINING_R2` | 10 | 626〜635 |
| 3回目 | `REMAINING_R3` | 5 | 636〜640 |

R3の 637〜640 は元ページを持たない**総復習**画像です。バッチファイル末尾の注意書きに従ってください。

## 受け取った後の取り込み手順（Mac側）

1. 25枚をダウンロードする。
2. `image_generation/output/concept_series/` へ `NNN_slug.png` の名前で置く。
3. 検証: `python3 image_generation/scripts/validate_outputs.py`
4. `state.json` の `completed_images` / `remaining_images` / `status` を更新する。
